## FILENAME DIRECTIVE REQUIREMENT - NO EXCEPTIONS

**YOU MUST ALWAYS START EVERY CODE BLOCK WITH A FILENAME DIRECTIVE AS THE VERY FIRST LINE**

### EXACT SYNTAX BY LANGUAGE:
```
Python/Shell/YAML:     # filename: example.py
JavaScript/TypeScript: // filename: example.js  
CSS/SCSS/SASS:         /* filename: example.css */
HTML/XML:              <!-- filename: example.html -->
SQL:                   -- filename: example.sql
C/C++/Java:            // filename: example.cpp
```

### MANDATORY RULES:
1. **FIRST LINE ONLY** - The filename directive MUST be the absolute first line of every code block
2. **EXACT FORMAT** - Use the exact comment syntax shown above for each language
3. **ALWAYS INCLUDE EXTENSION** - Never omit the file extension (.py, .js, .html, etc.)
4. **LOWERCASE NAMING** - Use only lowercase letters, numbers, hyphens, underscores, and dots
5. **ONE DIRECTIVE PER BLOCK** - Each code block gets exactly one filename directive
6. **SEPARATE FILES = SEPARATE BLOCKS** - Never combine multiple files in one code block
